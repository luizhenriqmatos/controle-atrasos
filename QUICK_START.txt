```bash
# 2. Fazer push para GitHub

```

**Resultado:** URL tipo `https://seu-projeto.netlify.app`

---

### 🎯 Opção Mais Comum: GitHub Pages (5 minutos)

```bash
# 1. Instalar gh-pages
npm install --save-dev gh-pages

# 2. Adicionar ao package.json:
# "homepage": "https://seu-usuario.github.io/controle-atrasos",
# "deploy": "gh-pages -d dist"

# 3. Deploy
npm run build
npm run deploy

# Pronto!
```

**Resultado:** URL tipo `https://seu-usuario.github.io/controle-atrasos`

## ❓ FAQ Rápido

**P: Posso usar a senha "admin123"?**
R: Sim, para desenvolvimento. Para produção, use hash bcrypt em `.env.local`

**P: Meus dados são salvos?**
R: Sim, em localStorage do navegador. Use "📥 Exportar Dados" para backup.

**P: Posso fazer deploy gratuitamente?**
R: Sim! Netlify ou GitHub Pages (ambos grátis com deploy automático)

**P: Preciso de backend?**
R: Não, tudo funciona offline em localStorage. Backend é opcional.

**P: Como resetar dados?**
R: Modo Admin → "🚨 Limpar Tabela" (com confirmação)

**P: Esqueci a senha?**
R: Sem recuperação (design intencional). Use DevTools para limpar localStorage.

## 📞 Suporte Rápido

### Erro: "npm: command not found"
→ Instalar Node.js em https://nodejs.org

### Erro: "Cannot find module"
→ Executar `npm install`

### Erro: "Port 5173 already in use"
→ Executar `npm run dev -- --port 5174`

### Dados sumiram
→ Verificar localStorage em DevTools (F12)
→ Recuperar com backup JSON (se tiver)
